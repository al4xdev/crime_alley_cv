# Karen Guard (Evaluator)

Isolated candidate evaluator. Simulates a highly skeptical Senior Technical Recruiter.

---

## How to Run

### Via Docker (Recommended — Total Isolation)

Build and execute the evaluation environment using the session ID generated by the orchestrator (Harvey):

```bash
cd karen_guard
./run.sh <session_id>
```

---

## Input Workspace Files

The evaluator expects the following files inside the session folder (`/app/session/docs`):

| File | Description |
|---|---|
| `cv.md` | Candidate's resume in markdown format |
| `job.md` | Job description and requirements |
| `who_are_u.md` | Context or briefing information |

---

## Agent Output

The evaluation process writes a markdown report (`evaluation.md`) to the session folder, which contains:

1. **Overview & Fit Score**: Rigorous fit score (0 to 100) and executive summary.
2. **Evidence Dossier**: Analysis of code quality, stacks, and Git hygiene in cloned repositories.
3. **Resume Validation**: Mapping confirmed claims vs inconsistencies or exaggerations.
4. **Red Flags**: Attention points regarding candidate credentials or practices.
5. **Interview Playbook**: Custom technical validation questions targeting code choices.
6. **Adjustment Recommendations**: Technical improvements for repositories and resume revision suggestions.
